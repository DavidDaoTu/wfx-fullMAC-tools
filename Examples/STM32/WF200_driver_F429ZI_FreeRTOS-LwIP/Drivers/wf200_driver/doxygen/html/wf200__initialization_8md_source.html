1 Initialization and configuration {#wf200_initialization}
2 ============
3 
4 It is recommended to reset WF200 before running the initialization phase described below.
5 Initialization and configuration is performed by ::wf200_init. WF200 initialization phase is as follows:
6 * Bus configuration (SPI or SDIO)
7 * WF200 initialization
8 * Firmware download
9 * Receive startup indication
10 * PDS configuration
11 
12 ## 1. Bus configuration
13 The bus configuration is handled by ::wf200_init_bus. It has a specific implementation depending on the bus used.
14 
15 ###SPI bus
16 
17 During the SPI initialization, the host follows the steps below:
18 * Reset WF200
19 * Read the \ref config_register "configuration register" to make sure the communication with WF200 is available.
20 * Write the \ref config_register "configuration register" with the SPI bus configuration settings preferred (e.g. word mode, high frequency SPI enabled...).
21 
22 ###SDIO bus
23 
24 During the SPI initialization, the host follows the steps below:
25 * Send SDIO command 0 \warning It is required for the host to wait a response from this command before issuing another command.
26 * Send SDIO command 8
27 * Send SDIO command 3
28 * Send SDIO command 7
29 * Enable function 1/cmd 53 in \ref CCCR_register "CCCR register".
30 * Enable interrupt.
31 * Configure the SDIO bus to function in 4-bit mode.
32 * Switch to high-speed mode (up to 50MHz).
33 * Set the block size used by the host in the \ref FBR_register "FBR register".
34 
35 ## 2. WF200 initialization
36 The WF200 initialization is handled by ::wf200_init_chip. During this phase, the host interacts with WF200 registers described in \ref wf200_registers.
37 * The first step performed is to set the correct values in the General purpose registers. Those values are linked to the crystal used by the hardware.
38 * Set the wake-up bit (bit 12) in the \ref control_register "control register".
39 * Wait for the wlan_rdy bit (bit 13) to be set in the \ref control_register "control register".
40 * Check if the access mode bit (bit 10) in the \ref config_register "configuration register" is at 1 (direct access mode).
41 
42 ## 3. Firmware download
43 
44 First, the WF200 clock is enabled in ::wf200_download_run_bootloader and an optional test to verify the SRAM access is performed.
45 
46 The firmware download is handled by ::wf200_download_run_firmware.
47 A firmware binary needs to be loaded to the chip at power-up. The binary is signed and encrypted. This binary file is stored in a c table in wfm_wf200_XX.h (XX depending on the keyset used by WF200).
48 Below is a diagram listing the steps followed by ::wf200_download_run_firmware to load the firmware.
49 \msc
50  hscale = "1";
51 
52  a [label="Host"],b [label="WF200"];
53 
54  a box a [label = "READY"];
55  b box b [label = "NOT READY"];
56  b=>a [ label = "INFO_READY"];
57  a=>b [ label = "INFO_READ"];
58  b=>a [ label = "READY"];
59  a=>b [ label = "UPLOAD_PENDING"];
60  b box b [label = "DOWNLOAD_PENDING"];
61  b box b [label = "DOWNLOAD_COMPLETE"];
62  b=>a [ label = "AUTH_OK"];
63  a=>b [ label = "OK_TO_JUMP"];
64 \endmsc
65 
66 After the host notifies the "OK_TO_JUMP" state, the WF200 will issue a startup indication when ready.
67 
68 ## 4. Startup indication
69 If the initialization process has been successful up to now the host should receive the startup indication from WF200.
70 
71 The start up indication is described in the ::HiStartupIndBody_t structure.
72 
73 ## 5. PDS configuration
74 Once the startup indication received, the host can send to WF200 the **PDS configuration** (Platform Data Set). The PDS contains information regarding the WF200 environment. Below are some examples:
75 * IO configuration
76 * XTAL tune
77 * TX power control
78 * RF output configuration
79 * ...
80 
81 The PDS is sent using a dedicated function ::wf200_send_configuration.
82 The PDS presents itself in the FMAC driver as several string tables found in wf200_pds.c. Those tables are the results of the compression of a more complete and readable file.
83 You can find more information on the PDS in a dedicated page \ref wf200_pds.
